# Security Policy

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability in ViVeTool GUI, please report it privately to help us address the issue before it can be exploited.

### How to Report

1. **GitHub Security Advisories (Preferred):** Use [GitHub's private vulnerability reporting](https://github.com/mta1124-1629472/ViVeTool-GUI/security/advisories/new) to submit a confidential report.

2. **Email:** If you prefer email, please reach out to the maintainers directly. You can find contact information in the repository's profile or through GitHub.

### What to Include

When reporting a vulnerability, please include:

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact of the vulnerability
- Any suggested fixes or mitigations (if available)

### Response Timeline

- We will acknowledge receipt of your report within **72 hours**
- We will provide an initial assessment within **7 days**
- We will work with you to understand and resolve the issue

## Supported Branches

| Branch   | Supported          |
| -------- | ------------------ |
| master   | :white_check_mark: |

Security updates will be applied to the `master` branch. Users are encouraged to use the latest release.

## Security Best Practices

When using ViVeTool GUI:

- Download releases only from the official GitHub repository
- Verify file integrity when possible
- Be cautious when enabling or disabling Windows features, as this can affect system stability
- Run with appropriate privileges only when necessary

## Acknowledgments

We appreciate responsible disclosure and will acknowledge security researchers who help improve the security of ViVeTool GUI (with their permission).
